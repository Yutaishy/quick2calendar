# Quick2Calendar Privacy Policy (Draft)

Last updated: 2026-02-15

## 1. Overview
Quick2Calendar enables users to create calendar events in their own Google Calendar account from natural-language input.

## 2. Data We Access
- Google Calendar event write access via OAuth scope:
  - `https://www.googleapis.com/auth/calendar.events`
- Event content provided by the user:
  - title, start time, end time, location, description

## 3. How We Use Data
- To create events in the user-selected calendar.
- To show registration results and recent local history.
- To improve interpretation quality based on user-defined rules.

## 4. Data Storage
- OAuth tokens are stored in macOS Keychain.
- Non-secret app settings are stored locally in Application Support.
- We do not intentionally store OAuth tokens in plaintext logs.

## 5. Data Sharing
- We do not sell personal data.
- Data is sent only to required providers for app functionality:
  - Google Calendar API
  - Gemini API (when AI mode is enabled)

## 6. User Controls
- Users can disconnect Google integration in app settings.
- Disconnect attempts token revocation and removes local token storage.
- Users can request data deletion via the Data Deletion page.
